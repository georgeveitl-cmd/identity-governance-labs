# Service Account Governance

## Overview

Service accounts are non-human identities used by applications and automation systems. These accounts often operate outside normal identity governance processes and can create security risks if not properly managed.

## Risks

• Excessive permissions
• Passwords that never expire
• Lack of monitoring
• No defined owner

## Governance Controls

• Assign an accountable owner
• Limit permissions to least privilege
• Rotate credentials regularly
• Monitor authentication activity

## Monitoring Strategy

Organizations should track:

• Service account authentication patterns
• Privileged access usage
• Credential age
• Application dependencies
